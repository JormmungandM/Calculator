VCS - Version Control System
Система обеспечения "точек восстановления", к которым всегда можно
вернуть проект
Git Github, Gitlab
Mercurial Bitbucket

- сохраняются только изменения (сложение, удаление, изменение)
  (комиты – commit – архив изменений)
- НЕ принято включать в VCS
  = зависимости (пакеты, которые могут быть загружены после
      установка проекта)
  = результаты компиляции (EXE, DLL и т.п., в т.ч. промежуточные – OBJ, cash)
  = файлы баз данных (??)
  = временные файлы, в т.ч. файлы-конфиги IDE ("студии") - .vs, .idea
  == все исключения прописываются в специальном файле .gitignore
      файл должен быть ориентирован на именно ту систему, с которой идет работа
       ("студия", язык программирования и т.д.) - универсального файла не существует
       https://github.com/github/gitignore – коллекция типовых файлов
     файл .gitignore может быть в каждой папке и действовать только на нее
      (включая подкаталогы)

Начало работы с VCS, как правило, происходит по схемам:
 1. Существует проект на локальном ПК, его нужно перенести на Git
 2. Существует проект на Git, его нужно скопировать на локальный ПК


1:
 - убедиться, что на ПК установлены средства git (https://git-scm.com/)
 - либо средствами "студии", либо через консоль (командная строка)
   = git init это создаст в каталоге скрытую папку .git
 - вносим изменения в код (работаем над проектом)
 - регистрируем эти изменения
   = git add ./ добавить текущую папку (все файлы)
   = git commit -m "The message" создаем комит (локально)
   = git push --all отправить в удаленный репозиторий
 - конфликты возникают когда изменения сделаны одновременно и при
    внесении изменений с одного ПК не учитываются изменения, сделанные
    с другой
